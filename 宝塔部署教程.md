# 🐷 猪朋友 - 宝塔面板部署教程

## 📋 前置要求

- 已安装宝塔面板的京东云服务器
- 服务器系统：CentOS 7+ / Ubuntu 18+ / Debian 9+
- 内存：至少 512MB（推荐 1GB+）

## 🚀 详细部署步骤

### 一、安装宝塔面板（如果还没有）

#### CentOS 系统
```bash
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec
```

#### Ubuntu/Debian 系统
```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484bec
```

### 七、配置 Nginx 反向代理

#### 1. 添加网站

点击【网站】→ 点击【添加站点】

**重要提示**：这是一个 Node.js 项目，请按以下方式选择：

填写信息：
- **域名**：你的域名（如 `pigfriends.com`）或服务器IP
- **根目录**：`/www/wwwroot/PigFriends/public`
- **PHP版本**：**选择"纯静态"** （不需要PHP，因为这是 Node.js 项目）
- **数据库**：**不创建数据库**（本项目使用 SQLite，不需要 MySQL/PostgreSQL）
- **FTP**：根据需要选择（可选）
- 其他保持默认

**为什么选择"纯静态"？**
- 本项目是 Node.js + Express 应用，不运行 PHP
- 静态文件（HTML/CSS/JS）由 Nginx 直接提供
- 后端 API 通过 Nginx 反向代理到 Node.js 服务（端口 3000）
- 数据存储使用 SQLite 文件数据库，无需 MySQL/PostgreSQL

点击【提交】

#### 2. 配置 Nginx 反向代理（使用伪静态方式）

由于宝塔面板可能没有反向代理选项，我们可以使用**伪静态配置**或**直接修改网站配置文件**来实现。

##### 方式一：使用伪静态配置（推荐，最简单）

1. 在网站列表中，找到刚才添加的站点
2. 点击【设置】
3. 点击左侧【伪静态】
4. 在配置框中，**清空现有内容**，然后粘贴以下配置：

```nginx
# 将所有 API 请求代理到 Node.js 服务（端口 3000）
location /api/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_cache_bypass $http_upgrade;
    
    # 支持大文件上传（base64图片，最大10MB）
    client_max_body_size 10m;
    proxy_read_timeout 300s;
    proxy_connect_timeout 300s;
}
```

5. 点击【保存】

**说明**：
- 这个配置只需要添加 API 代理规则
- 静态文件（HTML/CSS/JS）会自动由 Nginx 从 `/www/wwwroot/PigFriends/public` 目录提供（因为添加网站时已设置根目录）
- 如果你的网站根目录设置正确，这个简单配置就足够了

##### 方式二：直接修改网站配置文件（如果方式一不生效）

如果伪静态配置不生效或需要更精细的控制，可以直接修改网站配置文件：

1. 在网站列表中，找到你的站点，点击【设置】
2. 点击左侧【配置文件】
3. 找到 `server { ... }` 块，在现有的 `location / { ... }` 配置**之前**添加以下配置：

```nginx
    # API 请求代理到 Node.js 服务
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_cache_bypass $http_upgrade;
        
        # 支持大文件上传（base64图片，最大10MB）
        client_max_body_size 10m;
        proxy_read_timeout 300s;
        proxy_connect_timeout 300s;
    }
```

4. 确保 `location / { ... }` 配置指向正确的根目录：

```nginx
    location / {
        root /www/wwwroot/PigFriends/public;
        index index.html index.htm;
        try_files $uri $uri/ /index.html;
    }
```

**注意**：如果网站配置文件中的 `root` 已经设置为 `/www/wwwroot/PigFriends/public`，则无需修改 `location /` 配置。

5. 点击【保存】
6. **重要**：点击【测试配置】按钮，确认配置语法正确
7. 如果测试通过，点击【重载配置】使配置生效

##### 方式三：验证配置是否生效

配置完成后，可以通过以下方式验证：

1. **检查 Nginx 配置语法**：
   - 在网站设置 → 配置文件页面，点击【测试配置】
   - 或使用终端命令：`nginx -t`

2. **测试 API 代理**：
   - 打开浏览器访问：`http://你的域名/api/stats`
   - 应该返回 JSON 数据（统计信息）
   - 如果返回 502 错误，检查 Node.js 服务是否运行（PM2 管理器）

3. **测试静态文件**：
   - 打开浏览器访问：`http://你的域名/`
   - 应该能看到猪朋友首页
   - 打开浏览器开发者工具（F12），查看 Network 标签，确认静态文件（CSS/JS）正常加载

4. **测试完整功能**：
   - 尝试画一只猪并提交
   - 查看是否能正常显示所有猪的列表
   - 测试点赞功能

**配置说明**：
- `/api/*` 路径的所有请求都会被代理到 Node.js 服务（端口 3000）
- 静态文件（HTML/CSS/JS/图片）由 Nginx 直接从 `public` 目录提供
- `X-Forwarded-Proto $scheme` 确保 SSL 模式下正常工作（启用 HTTPS 后自动处理）
- `client_max_body_size 10m` 支持上传大图片（base64格式，最大10MB）
- `proxy_read_timeout` 和 `proxy_connect_timeout` 设置较长的超时时间，避免大文件上传超时

#### 3. 测试访问

**配置 SSL 前**：
打开浏览器访问：
- `http://你的域名` 或 `http://你的IP`

**配置 SSL 后**：
- HTTP 访问：`http://你的域名` （会自动跳转到 HTTPS，如果开启了强制HTTPS）
- HTTPS 访问：`https://你的域名` （推荐）

应该能看到猪朋友首页！🎉

### 八、配置 SSL 证书（可选但推荐）

> **快速指南**：如果你已经申请了京东云 SSL 证书，按以下步骤部署：
> 1. 在京东云控制台下载 Nginx 格式证书（.crt 和 .key 文件）
> 2. 在宝塔面板 → 网站 → 设置 → SSL → 其他证书
> 3. 粘贴证书内容和私钥内容
> 4. 开启强制HTTPS
> 5. 验证访问 https://你的域名

---

#### 方式一：使用京东云 SSL 证书（推荐，已申请证书时使用）

##### 1. 下载证书文件

1. 登录【京东云控制台】
2. 进入【云安全】→【SSL数字证书】
3. 找到已申请的证书，点击【下载】
4. 选择下载格式：**Nginx** 格式
5. 下载后会得到一个压缩包，解压后包含：
   - `证书文件名.crt`（证书文件）
   - `证书文件名.key`（私钥文件）
   - 可能还有 `chain.crt`（证书链文件，如果有的话）

**证书文件说明：**
- `.crt` 文件：SSL 证书文件
- `.key` 文件：私钥文件（非常重要，请妥善保管）
- `chain.crt` 文件：证书链文件（如果有的话）

##### 2. 上传证书到服务器

**方法1：通过宝塔面板文件管理器上传**

1. 在宝塔面板中，点击【文件】
2. 进入 `/www/server/panel/vhost/cert/你的域名/` 目录（如果没有则创建）
3. 点击【上传】，上传以下文件：
   - 证书文件（`.crt`）
   - 私钥文件（`.key`）
   - 证书链文件（如果有 `.chain.crt` 或 `chain.crt`）

**方法2：通过 SSH 终端上传**

1. 使用 SCP 或 SFTP 工具将证书文件上传到服务器
2. 或直接在服务器上创建证书目录并上传：

```bash
# 创建证书目录（以域名 pigfriends.com 为例）
mkdir -p /www/server/panel/vhost/cert/pigfriends.com

# 使用文本编辑器创建证书文件（将证书内容粘贴进去）
nano /www/server/panel/vhost/cert/pigfriends.com/fullchain.crt
# 粘贴 .crt 文件内容，保存退出（Ctrl+X, Y, Enter）

nano /www/server/panel/vhost/cert/pigfriends.com/privkey.key
# 粘贴 .key 文件内容，保存退出

# 如果有证书链文件
nano /www/server/panel/vhost/cert/pigfriends.com/chain.crt
# 粘贴证书链内容，保存退出
```

##### 3. 在宝塔面板配置 SSL 证书

1. 在宝塔面板中，点击【网站】
2. 找到你的网站，点击【设置】
3. 点击左侧【SSL】
4. 选择【其他证书】
5. 填写证书信息：

   **证书内容（PEM格式）**：
   - 打开下载的 `.crt` 文件
   - 复制全部内容（包括 `-----BEGIN CERTIFICATE-----` 和 `-----END CERTIFICATE-----`）
   - 粘贴到【证书(PEM格式)】文本框
   
   **私钥（KEY格式）**：
   - 打开下载的 `.key` 文件
   - 复制全部内容（包括 `-----BEGIN PRIVATE KEY-----` 和 `-----END PRIVATE KEY-----`）
   - 粘贴到【私钥(KEY格式)】文本框
   
   **证书链（可选，如果有 chain.crt 文件）**：
   - 如果有证书链文件，打开 `chain.crt` 文件
   - 复制全部内容
   - 粘贴到【证书链(可选)】文本框

6. 点击【保存】

##### 4. 开启强制 HTTPS

1. 在 SSL 设置页面，找到【强制HTTPS】开关
2. 开启【强制HTTPS】
3. 点击【保存】

##### 5. 验证 SSL 配置

1. 打开浏览器访问：`https://你的域名`
2. 确认浏览器地址栏显示锁图标 🔒
3. 点击锁图标，查看证书信息，确认证书有效期

**如果显示证书错误，请检查：**
- 证书文件和私钥文件内容是否完整
- 证书域名是否与访问域名匹配
- 443 端口是否已开放（防火墙和安全组）
- Nginx 配置是否正确

### 十、数据备份设置（重要！）

#### 1. 自动备份数据库文件

2. 点击【添加计划任务】

配置：

- **任务名称**：备份猪朋友数据库

- **脚本内容**：

```bash
#!/bin/bash

fi
```

```bash
mkdir -p /www/backup

```

## ⚠️ 常见问题

### 1. 访问提示 502 错误

**原因**：Node.js 服务未启动

**解决**：
1. 打开 PM2 管理器
2. 检查项目状态是否为"运行中"
3. 如果已停止，点击【启动】

### 2. 无法上传大图片

**原因**：Nginx 上传文件大小限制

**解决**：
1. **如果已使用伪静态配置**：
   - 伪静态配置中已包含 `client_max_body_size 10m;`
   - 确认配置已保存并重载 Nginx

2. **如果仍然无法上传**：
   - 网站设置 → 配置文件
   - 在 `server { ... }` 块中添加（如果还没有）：
     ```nginx
     client_max_body_size 10m;
     ```
   - 或者在 `http { ... }` 块中添加（全局设置）
   - 保存并重载配置

3. **检查 Express 配置**：
   - 确认 `server/server.js` 中已设置 `express.json({ limit: '10mb' })`
   - 这个已经在代码中配置好了

### 3. 端口被占用

**解决**：
```bash
# 查看端口占用
netstat -tunlp | grep 3000
# 或使用宝塔面板【文件】→【端口管理】
```

修改项目端口：
1. 编辑 `server/server.js`
2. 修改 `PORT` 变量
3. 同时修改 Nginx 反向代理的目标端口

### 4. 数据库文件权限问题

**解决**：
```bash
cd /www/wwwroot/PigFriends
chmod 644 pigs.db
chown www:www pigs.db
```

### 5. Node.js 版本不对

**解决**：
1. 打开 Node.js 版本管理器
2. 安装推荐版本（v18.x 或 v20.x）
3. 在 PM2 管理器中切换 Node 版本

### 6. SSL 证书配置后无法访问 HTTPS

**原因**：可能是证书配置错误或端口未开放

**解决**：
1. **检查证书格式**：
   - 确保证书内容完整（包括 `-----BEGIN CERTIFICATE-----` 和 `-----END CERTIFICATE-----`）
   - 确保私钥内容完整（包括 `-----BEGIN PRIVATE KEY-----` 和 `-----END PRIVATE KEY-----`）
   - 证书和私钥内容不要有多余的空格或换行

2. **检查域名匹配**：
   - 确保证书域名与访问域名完全一致
   - 如果使用通配符证书（如 `*.example.com`），确保格式正确

3. **检查端口开放**：
   - 确保宝塔面板防火墙已开放 443 端口
   - 确保京东云安全组已开放 443 端口（HTTPS）

4. **检查 Nginx 配置**：
   - 在网站设置 → 配置文件，查看 SSL 配置是否正确
   - 确认 `listen 443 ssl;` 配置存在
   - 重启 Nginx：`nginx -t && nginx -s reload`

5. **查看错误日志**：
   - 宝塔面板 → 网站 → 设置 → 日志
   - 查看错误日志了解具体错误信息

### 7. SSL 证书显示"不安全"或"证书错误"

**原因**：证书链不完整或证书过期

**解决**：
1. **检查证书链**：
   - 如果有 `chain.crt` 文件，确保已添加到【证书链(可选)】文本框
   - 或者将证书链内容合并到证书文件中

2. **检查证书有效期**：
   - 在浏览器中点击锁图标，查看证书有效期
   - 如果证书已过期，需要重新申请并上传新证书

3. **清除浏览器缓存**：
   - 清除浏览器缓存和 SSL 状态
   - 或使用隐私模式访问测试

### 8. 强制 HTTPS 后无法访问网站

**原因**：可能是证书配置有问题或 443 端口未开放

**解决**：
1. 暂时关闭【强制HTTPS】开关
2. 先测试 HTTP 访问是否正常
3. 检查 SSL 证书配置是否正确
4. 确保 443 端口已开放
5. 配置正确后，再开启【强制HTTPS】

### 9. API 请求返回 404 或 502 错误

**原因**：Nginx 反向代理配置不正确或 Node.js 服务未运行

**解决**：
1. **检查 Node.js 服务是否运行**：
   - 打开 PM2 管理器，确认 `pig-friends` 项目状态为"运行中"
   - 如果未运行，点击【启动】

2. **检查伪静态配置**：
   - 确认伪静态配置中已添加 `/api/` 的 location 配置
   - 确认 `proxy_pass http://127.0.0.1:3000;` 地址正确
   - 确认端口 3000 与 PM2 中配置的端口一致

3. **检查网站配置文件**：
   - 如果使用方式二（直接修改配置文件），确认配置已正确添加
   - 确认配置语法正确（使用【测试配置】功能）
   - 确认已重载 Nginx 配置

4. **测试 API 连接**：
   ```bash
   # 在服务器终端测试 Node.js 服务是否正常
   curl http://127.0.0.1:3000/api/stats
   ```
   - 如果返回 JSON 数据，说明 Node.js 服务正常
   - 如果连接失败，检查 PM2 日志

5. **查看 Nginx 错误日志**：
   - 宝塔面板 → 网站 → 设置 → 日志
   - 查看错误日志了解具体错误信息

### 10. 静态文件无法加载（CSS/JS 404 错误）

**原因**：网站根目录配置不正确

**解决**：
1. **检查网站根目录设置**：
   - 网站设置 → 网站目录
   - 确认根目录为：`/www/wwwroot/PigFriends/public`
   - 如果不对，修改为正确的路径

2. **检查文件权限**：
   ```bash
   # 确保 public 目录有读取权限
   chmod -R 755 /www/wwwroot/PigFriends/public
   chown -R www:www /www/wwwroot/PigFriends/public
   ```

3. **检查文件是否存在**：
   ```bash
   # 确认文件存在
   ls -la /www/wwwroot/PigFriends/public/
   ```

4. **清除浏览器缓存**：
   - 按 `Ctrl + F5` 强制刷新
   - 或使用隐私模式访问测试

### 11. 伪静态配置不生效

**原因**：配置格式错误或 Nginx 未重载

**解决**：
1. **检查配置语法**：
   - 在伪静态配置页面，点击【保存】后，查看是否有错误提示
   - 或使用【配置文件】→【测试配置】功能

2. **确认配置位置**：
   - 伪静态配置会插入到 `location / { ... }` 之前
   - 如果网站配置文件中有多个 `location` 块，确认配置顺序正确

3. **重载 Nginx**：
   - 修改配置后，必须点击【重载配置】或重启 Nginx
   - 在网站设置 → 配置文件页面，点击【重载配置】

4. **使用方式二**：
   - 如果伪静态配置不生效，改用方式二（直接修改网站配置文件）
   - 在配置文件中直接添加 location 块

## 📊 性能优化建议

### 1. 开启 Gzip 压缩

网站设置 → 配置文件，添加：
```nginx
gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;
gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss;
```

### 2. 配置静态资源缓存

网站设置 → 配置文件，添加：
```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 7d;
    add_header Cache-Control "public, no-transform";
}
```

### 3. 限制访问频率（防刷）

安装【Nginx防火墙】插件，配置 CC 防护
